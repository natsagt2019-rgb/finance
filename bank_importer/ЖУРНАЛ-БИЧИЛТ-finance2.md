> finance 2.0-д ачаалсан **бодит дансны кодоор** (Түмэн Тээх E-balance чарт) буулгасан.
> ⚠️ Reference дахь `АМЬД (accounting.db)` ба `2026 (Excel)` код хоёрын **аль нь ч** энд хүчингүй —
> finance 2.0 бол гурав дахь схем. Доорх код л бодит (амьд DB дээр баталгаажсан).

## 0. finance 2.0 чартын онцлог (заавал ойлгох)

| Зүйл | finance 2.0 дахь хэрэгжилт |
|---|---|
| **70xxxx ↔ 71xxxx ХОС цуваа** | Ижил нэртэй 2 зэрэгцээ мод. `70xxxx`=**үйл ажиллагааны зардал** (P&L "Зардал"), `71xxxx`=**борлуулалтын өртөг/ББӨ** (P&L "Өртөг"). Гүйлгээний шинжээр сонгоно: тээвэртэй шууд→71xx, захиргаа→70xx. |
| **6xxxxx = борлуулалтын өртөг** | `610201 Үзүүлсэн ажил үйлчилгээний өртөг`, `610101 БҮ-ний өртөг`. ББӨ-д тооцогдоно. |
| **ББӨ таних** | `is_cogs` флаг ХООСОН. ББӨ = код prefix `6xxxxx` ЭСВЭЛ `71xxxx`. (pnl-summary, тайлан энэ дүрмээр.) |
| **Тайлан мэплэлт** | Кодын prefix-д биш — `accounts.fs_line` (СБТ/ОДТ мөр) + `accounts.type`-аар. |
| **Эх сурвалж** | Бүх тайлан `journal_entries`-ээс. Модуль → `postJournal()` → `mirrorToLedger` → journal_entries. |

### ⭐ Түмэн Тээхийн ГОЛ ӨРТӨГ (хамгийн чухал)
Компанийн орлогын эх үүсвэр = тээврийн үйлчилгээ, гол өртөг нь **дэд гүйцэтгэгчээр тээвэрлүүлсэн төлбөр** (банкинд "машин түрээс", "кран түрээс", "Хайрга/нүүрс тээврийн төлбөр" гэж харагдана). 2026 онд энэ нь **3.3 тэрбум** (ангилал `1.2.1`+`1.2.2`).
- **Өртгийн данс = `711701 Тээврийн зардал`** (ББӨ, 71xx цуваа). *(Шийдвэр: 610201 биш 711701.)*
- **Бичилт (аккруал):** ① Дт `711701` / Кт `310101 Дансны өглөг` (тээвэр авах) → ② Дт `310101` / Кт банк (төлөх).
- ⚠️ Одоогийн 2026 журнал `1.2.1`/`1.2.2`-г **610201**-д, cash-basis шууд бичсэн — **711701 рүү remap + өглөгөөр** засах ёстой.
- **Орлого ← нэхэмжлэхээс:** Дт `120101` / Кт `510102` + Кт `310601` НӨАТ (нэхэмжлэх НӨАТ-тай бол).

### ⭐ К-код = харицсан менежер (зардлын төв)
Гүйлгээний утгын эхэнд байгаа `K1`–`K10` нь **хариуцсан хүн (менежер)**. Банкны зардал ↔ нэхэмжлэхийн `responsible` талбар ↔ менежерийг холбож, **менежер бүрийн ашигт ажиллагаа** (орлого−өртөг) гаргана. Журналд энэ хэмжүүрийг хадгалбал cost-center P&L гарна.

| К | Менежер | | К | Менежер |
|---|---|---|---|---|
| K1 | Отгонбаатар | | K6 | Эрдэнэтуяа |
| K2 | Нямдорж | | K7 | Уранзаяа |
| K3 | Баяраа | | K8 | Одонтунгалаг |
| K4 | Чингүүн | | K9 | Амар |
| K5 | Алтансүх | | K10 | Тэргэл |

> Банк утгаас салгах: `regexp_match(upper(description), '^K(10|[1-9])')`. Нэхэмжлэх дээр `responsible`=менежерийн нэр.

**Дансны үндсэн бүлгүүд:**
`100101` касс · `1101xx` банкууд (Голомт 110101 / ХХБ 110102 / М банк 110103) ·
`12xxxx` авлага · `180xxx` урьдчилж төлсөн · `20xxxx` ҮХ өртөг · `201xxx` хуримт.элэгдэл ·
`31xxxx` өр төлбөр/татвар/цалин · `320101` урьдчилж олсон орлого · `43xxxx` өмч ·
`51xxxx` орлого · `6xxxxx`/`71xxxx` өртөг(ББӨ) · `70xxxx` үйл ажиллагааны зардал ·
`84xxxx` бусад орлого(хүү/ханш) · `85xxxx` ханшийн олз · `87/88xxxx` бусад зардал/ханшийн гарз.

## Гол дүрэм (8)
1. **Аккруал** — орлого: авлага→банк; зардал: өглөг→банк. Банктай шууд бүү хос.
2. **Дт = Кт** хадгалахаас өмнө.
3. **НӨАТ тусгай данс** — input `120201`, output `310601`. Зардал/орлогод бүү нийлүүл.
4. **Хуримтлуулах vs Төлөх** — цалин/хөлс эхлээд өглөг, дараа банк.
5. **ББӨ (71xx/6xxx) vs Үйл ажиллагаа (70xx)** — гүйлгээний шинжээр сонго.
6. **Урьдчилгаа** — захиалагчийнх `320101` (өр); ажилтных `120601` (хөрөнгө).
7. **ҮХ-ийг өртөгөөр** (`20xxxx`), элэгдлээр шингээ — шууд зардалд бүү бич.
8. **draft → posted** — батлагдсан үед л тайланд.
